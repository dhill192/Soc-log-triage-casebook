# SOC Log Triage

A cybersecurity portfolio project focused on triaging and classifying suspicious log events in a Tier 1 SOC-style workflow.

---

## Overview

This project simulates the first-pass review process a SOC analyst performs when investigating log activity.

It is designed to help identify, organize, and prioritize suspicious events for further investigation.

---

## What This Project Demonstrates

- Basic security event triage
- Log review and classification
- Identifying suspicious patterns in authentication or system logs
- Separating benign activity from potentially malicious behavior
- Security-focused scripting and investigation workflow

---

## Analyst Workflow Simulated

1. Review raw log entries
2. Parse relevant fields
3. Identify unusual or suspicious behavior
4. Group or classify findings
5. Prepare results for escalation or further review

---

## Example Use Cases

This project is intended to reflect tasks related to:

- Failed login review
- Repeated authentication attempts
- Suspicious IP behavior
- Initial alert validation
- Tier 1 SOC investigation support

---

## SOC Relevance

This project demonstrates practical skills relevant to entry-level SOC work, including:

- Reading and interpreting logs
- Recognizing suspicious activity
- Organizing findings clearly
- Thinking in terms of triage and escalation
